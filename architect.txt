1. Authentication
    When an client signup, the backend send back to client the access token and refresh token and create new User record in database.
    Client -> POST /api/users/signup
        Request Body:
        {
            "username": "...",
            "password": "...",
            "email": "..."
        }
    Server -> Client
        Response:
        {
            "access_token": "...",
            "refresh_token": "...",
        }
    
    When an client signin, the backend verify username, password and otp and then change the status of client to online and send back to client the user data from database.
    Client -> GET /api/users/signin
        Request Body:
        {
            "username": "...",
            "password": "...",
            "otp": "..."
        }
        Request Auth: Bearer Token
        token: access_token
    Server -> Client
        Response:
        {
            ...
        }
